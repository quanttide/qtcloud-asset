# AI 执行审核契约 — 使用示例

## 示例场景

**场景**：AI 助手需要执行删除文件操作，需要用户审核。

---

## 对话示例

### 用户请求

```
帮我删除 src/cli/tests/ 目录下的一些旧测试文件
```

### AI 读取契约

AI 自动读取 `.quanttide/ai/contract.yaml`，识别到"删除文件"是需要审核的操作。

### AI 列出操作清单

```
我需要执行以下操作：

1. 删除 src/cli/tests/old_test.py（需审核）
2. 删除 src/cli/tests/test_backup.py（需审核）

以上操作涉及"删除文件"，属于需要审核的高风险操作。

请确认是否执行？
```

### 用户确认

```
可以执行
```

### AI 执行

```
执行删除操作...
- 已删除 src/cli/tests/old_test.py
- 已删除 src/cli/tests/test_backup.py

完成。
```

## 示例 3：混合操作

### 用户请求

```
帮我重构 src/cli/app/ 目录，把所有函数改用 async 语法
```

### AI 分析

AI 读取契约，发现这次请求包含多个操作：
- 修改现有文件（不需要审核）
- 如果有测试文件，需要运行测试

### AI 列出操作

```
我将执行以下操作：

1. 修改 src/cli/app/cli.py（无需审核）
2. 修改 src/cli/app/workflow.py（无需审核）
3. 修改 src/cli/app/contract.py（无需审核）
4. 运行测试 pytest（无需审核）

以上操作均为修改和测试类操作，无需额外审核。

开始执行...
```

## 示例 4：用户拒绝

### 用户请求

```
帮我删除整个 src/ 目录
```

### AI 分析

AI 读取契约，发现这是"rm -rf"级别的破坏性操作。

### AI 拒绝执行

```
此操作涉及：
- 删除整个 src/ 目录（需审核）
- rm -rf 级别的破坏性操作（需审核）

此操作风险极高，我无法执行。

如果您确实需要执行此操作，请：
1. 确认这是预期行为
2. 备份重要数据
3. 手动在终端执行
```

## 示例 5：Git push 操作

### 用户请求

```
帮我提交并推送到远程
```

### AI 分析

AI 读取契约，发现"git push"是需要审核的操作。

### AI 列出操作

```
我将执行以下操作：

1. git add -A（无需审核）
2. git commit -m "feat: 更新功能"（无需审核）
3. git push origin main（需审核）

请确认是否执行 git push？
```

### 用户确认

```
可以
```

### AI 执行

```
执行 git push...
已推送到 origin/main

完成。
```
