# 交互设计文档

## 设计原则

### 极简主义

每个界面只做一件事。用户进入界面后 3 秒内知道能做什么。隐藏不必要的选项，高级功能通过二级入口访问。

### 一致性

相同操作在不同页面表现一致。按钮位置、颜色、文案保持统一。导航结构全局一致。

### 反馈即时

用户操作后立即给出反馈。加载状态明确，成功失败都有提示。不让用户猜测系统是否在运行。

## 核心界面

### 资产全景图

**图谱视图**

左侧是图谱画布，占据 80% 宽度。右侧是详情面板，占据 20% 宽度，默认隐藏，点击节点后滑出。

画布支持鼠标滚轮缩放、拖拽平移。节点用不同颜色区分资产类型：文档蓝色、代码绿色、示例橙色、流程紫色。连线粗细表示关系强度，虚线表示手动关系，实线表示自动发现关系。

点击节点右侧面板滑出，显示资产名称、类型、仓库、修改时间、关联列表。关联列表可点击跳转到对应节点。

**列表视图**

表格展示所有资产。列包括名称、类型、仓库、修改时间、关联数。支持点击列头排序，支持按类型和仓库筛选。

行点击打开资产详情。行双击跳转到图谱视图并定位到该节点。

顶部有切换按钮在图谱视图和列表视图之间切换。

**顶部导航**

面包屑显示当前位置：首页 > 资产全景图。右侧有刷新按钮，触发手动同步。

### AI 交付约束

**规则列表页**

表格展示所有规则。列包括规则名称、适用类型、状态、最后验证时间、通过率。

每条规则有操作按钮：编辑、启用/禁用、删除。顶部有新建按钮。

**规则编辑页**

表单包含：规则名称、描述、适用资产类型（下拉选择）、验证条件列表、严重级别。

验证条件支持动态添加和删除。每条条件包含类型选择（格式检查/内容检查/关联检查）、条件参数、阈值。

底部有保存和取消按钮。保存前可点击测试按钮，用示例资产验证规则是否正确。

**验证报告页**

顶部显示总分和通过状态。中间是各规则的验证结果列表，包含规则名、结果、失败原因、建议。

失败项高亮显示，可点击展开详细信息。底部有重新验证按钮。

### 数字资产定价

**定价报告页**

顶部选择器支持选择资产或资产组合。点击生成按钮后显示加载状态，完成后展示报告。

报告分三块：总资产估值卡片、估值明细表格、数据来源说明。

总资产估值卡片显示金额、置信度、估值日期。估值明细表格列出各资产的名称、估值、方法、置信度。数据来源说明列出使用的数据源和数据量。

底部有导出按钮，支持 PDF 和 Markdown 格式。

**价格数据管理页**

表格展示已采集的价格信号。列包括来源类型、金额、时间、状态（已确认/待审查）。

顶部有导入按钮，支持 CSV 文件导入。导入后显示校验结果，有问题数据提示审查。

## 交互流程

### 查看资产全景图

用户进入系统 -> 点击资产全景图 -> 默认展示图谱视图 -> 缩放查看细节 -> 点击节点看详情 -> 可切换到列表视图按条件筛选

### 验证 AI 交付物

智能体提交资产 -> 系统自动触发验证 -> 生成验证报告 -> 用户查看报告 -> 不通过则返回修改，通过则确认接收

### 生成定价报告

用户打开定价页面 -> 选择资产范围 -> 点击生成 -> 查看估值结果 -> 可导出报告
