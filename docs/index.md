# 工作文档

## 产品简介

### 资产全景

**场景**：一个项目积攒了上百份文档、几十个代码库、多个服务部署，东西散落在不同仓库、不同人手里。当需要判断项目完整度或接盘一个新模块时，没有一张能看清全貌的地图。

**困境**：资产之间的关联关系只存在于当事人的脑子里。人离职、交接不全，关系就断了。管理者无法准确判断研发进度、缺失哪些关键资产。

**救援**：资产云将分散的数字资产建模为可追溯的关系图——文档、代码、服务、数据的来源和依赖一目了然。资产变化自动记录，全景图随项目同步演化。

### AI 交付约束

**场景**：AI 智能体能在几分钟内产出设计文档、代码骨架和测试用例，速度远远超过人工。但内容质量参差不齐——有时它编造了一个不存在的 API，有时它忽略了关键边界条件。

**困境**：每次验收都需要有经验的人逐条审查细节，AI 写 5 分钟人审半小时。速度优势被验收成本抵消。而约束规则（如"API 必须与已有接口一致"）散落在人的经验里，无法规模化执行。

**救援**：资产云将约束规则显性化为可执行的验证契约——交付物提交后自动逐条校验，不合规的地方直接标出。人不再逐字审查，只关注 AI 标记的疑点，验收效率从小时级降到分钟级。

### 资产定价

**场景**：软件公司的核心资产是代码、数据、专利、客户关系，但这些东西在资产负债表上看不见。当需要融资、并购或技术转让时，拿什么证明这些数字资产值多少钱？

**困境**：交易数据分散在 Git 日志、CI/CD 指标、用户增长报表、合同归档里，缺乏系统化的收集和分析框架。定价谈判靠拍脑袋，要么低估吃亏，要么高估吓跑对方。

**救援**：资产云将数字资产的开发投入、使用频次、贡献规模等维度数据化，生成结构化的估值报告。定价不再靠直觉，而是基于可追溯的事实维度。
