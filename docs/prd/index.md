# 数字资产云蓝图

量潮数字资产云（qtcloud-asset）产品需求文档，聚焦三大核心能力的产品设计和功能规划。

## 核心能力

### 数字资产全景图

自动构建知识图谱，展示资产全景和关联关系。包含资产发现、关系抽取、可视化、动态同步、成熟度评分。

→ [查看详情](graph.md)

### AI 交付约束

定义验收规则，自动验证智能体交付物。包含规则定义、规则模板、验证引擎、报告反馈、规则版本管理。

→ [查看详情](harness.md)

### 数字资产定价

从交易过程提取价格信号，建立定价模型。包含价格采集、数据源适配、定价模型、报告分析。

→ [查看详情](pricing.md)

## 底层引擎

### 工作流引擎

契约的执行引擎，让定义的契约从静态文档变为可自动运行的跨平台流水线。

→ [查看详情](workflow.md)

## 已验证

- 本地脚本管理 Markdown 资产：通过 `backup_product_journal.py` 验证了资产的采集、移动、归档流程
- 本地 AI 转换管道：通过 `generate_product_roadmap.py` 验证了调用本地 Ollama 模型将日志转换为结构化蓝图的可行性
- 目录结构即资产模型：journal/archive/roadmap 三目录结构验证了"资产建模、版本快照、事件路由"的内核设计

## 困惑与挑战

1. 数据一致性 -> 通过最终一致性与状态机解决，为每个资产定义状态机（草稿 -> 已发布 -> 已归档），记录同步日志
2. 用户需求变化 -> 通过插件化架构解决，需求变化只需新增插件，内核不动
3. 格式丢失处理 -> V1.0严格限定资产边界，不碰复杂富文本，降低转换复杂度
