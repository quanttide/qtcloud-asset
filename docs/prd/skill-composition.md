# 原子技能编排

## 故事 1：查看原子技能库

### 用户故事

作为配置管理员，我想要浏览系统提供的所有原子技能，以便了解有哪些可用的积木可以组合。

### 验收标准

**场景**：查看技能列表

- **假设**：用户已登录系统
- **当**：用户打开技能库页面
- **那么**：系统显示所有原子技能的列表，每个技能显示名称、描述、输入参数、输出结果

**场景**：查看技能详情

- **假设**：用户已打开技能库页面
- **当**：用户点击某个技能的名称
- **那么**：系统展开显示该技能的完整描述、参数说明、使用示例

### 业务规则

- 技能库为只读视图，不可编辑
- 技能按类型分类展示（文件操作、权限操作、压缩操作等）
- 每个技能有唯一标识符用于配置引用

### 依赖

- 故事 2：定义工作流

## 故事 2：定义工作流

### 用户故事

作为配置管理员，我想要通过配置文件定义新工作流，以便灵活组合原子技能满足业务需求。

### 验收标准

**场景**：创建新工作流

- **假设**：用户已打开工作流配置页面
- **当**：用户填写工作流名称、选择技能、设置执行顺序
- **那么**：系统生成配置文件，包含工作流定义

**场景**：预览工作流

- **假设**：用户已定义工作流配置
- **当**：用户点击预览按钮
- **那么**：系统以可视化方式展示技能执行顺序，并验证配置正确性

**场景**：配置校验

- **假设**：用户提交工作流配置
- **当**：配置的技能引用了不存在的技能
- **那么**：系统显示错误提示，阻止保存

### 业务规则

- 工作流名称唯一，不可重复
- 至少包含一个技能
- 技能顺序可拖拽调整
- 支持为每个技能设置参数值
- 支持技能间的数据传递

### 依赖

- 故事 1：查看原子技能库

## 故事 3：执行工作流

### 用户故事

作为配置管理员，我想要执行已定义的工作流，以便自动化完成治理任务。

### 验收标准

**场景**：执行归档工作流

- **假设**：用户已定义"归档"工作流（列举文件 → 移动文件 → 验收结果）
- **当**：用户触发执行工作流
- **那么**：系统按顺序执行每个技能，完成后显示执行结果汇总

**场景**：预览执行

- **假设**：用户已定义工作流
- **当**：用户点击"预览执行"按钮
- **那么**：系统显示将要执行的操作列表，不实际执行

**场景**：执行中断

- **假设**：工作流正在执行中
- **当**：某个技能执行失败
- **那么**：系统停止执行，显示错误详情，已执行部分可回滚

### 业务规则

- 执行前先进行配置校验
- 支持同步和异步执行模式
- 执行过程可中断和取消
- 失败时自动回滚（可选配置）
- 执行记录持久化保存

### 依赖

- 故事 2：定义工作流

## 故事 4：管理执行记录

### 用户故事

作为配置管理员，我想要查看工作流的执行历史，以便追溯和审计。

### 验收标准

**场景**：查看执行历史

- **假设**：用户已执行过工作流
- **当**：用户打开执行历史页面
- **那么**：系统显示所有执行记录列表，包含时间、工作流名称、执行人、状态

**场景**：查看执行详情

- **假设**：用户已打开执行历史页面
- **当**：用户点击某条执行记录
- **那么**：系统显示该次执行的完整详情，包括每个技能的执行结果、耗时、输入输出

### 业务规则

- 执行记录保留 90 天
- 支持按工作流名称、执行状态、时间范围筛选
- 可导出执行报告

### 依赖

- 故事 3：执行工作流

## 故事 5：导入导出配置

### 用户故事

作为配置管理员，我想要导入导出工作流配置，以便在不同环境间迁移或备份。

### 验收标准

**场景**：导出配置

- **假设**：用户已定义工作流
- **当**：用户选择导出功能
- **那么**：系统生成配置文件供下载，包含完整的工作流定义

**场景**：导入配置

- **假设**：用户有工作流配置文件
- **当**：用户上传配置文件
- **那么**：系统解析并导入配置，冲突时提示处理方式

### 业务规则

- 导出格式为 JSON 或 YAML
- 导入时进行配置校验
- 冲突处理：覆盖、跳过、保留两者

### 依赖

- 故事 2：定义工作流

## 故事 6：原子技能注册

### 用户故事

作为开发者，我想要注册新的原子技能，以便扩展技能库。

### 验收标准

**场景**：注册技能

- **假设**：开发者已实现技能逻辑
- **当**：开发者提交技能注册信息
- **那么**：系统将新技能添加到技能库

**场景**：技能发现

- **假设**：新技能已注册
- **当**：配置管理员查看技能库
- **那么**：新技能出现在列表中，可用于配置工作流

### 业务规则

- 技能必须实现标准接口
- 技能必须有完整的参数和返回值定义
- 注册后需审核通过才能使用

### 依赖

无
