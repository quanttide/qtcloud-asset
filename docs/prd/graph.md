# 数字资产全景图

## 故事一：资产全景可视化

### 用户故事
作为查询方，我想要查看数字资产的全景图和关联关系，以便快速了解资产结构和依赖关系。

### 验收标准

**场景**：查看资产图谱
- **假设**：系统已扫描并识别仓库中的数字资产
- **当**：查询方打开全景图页面
- **那么**：系统显示资产节点和关联连线的可视化图谱

**场景**：筛选资产
- **假设**：图谱中有多种类型的资产
- **当**：查询方选择按资产类型筛选
- **那么**：图谱仅显示选中类型的资产节点

**场景**：查看资产详情
- **假设**：查询方在图谱中看到目标资产节点
- **当**：查询方点击该节点
- **那么**：系统显示资产名称、类型、仓库、修改时间、关联数量等详情

### 业务规则

- 节点大小表示资产的关联度，关联越多节点越大
- 连线粗细表示关系强度，引用越多连线越粗
- 支持的关系类型包括引用、依赖、继承、演变、包含
- 自动发现的关系与手动添加的关系用不同样式区分

### 依赖

- 依赖资产发现能力，需先扫描识别资产
- 依赖关系抽取能力，需先建立资产关联

## 故事二：资产变更追踪

### 用户故事
作为关注方，我想要在依赖的资产变更时收到通知，以便及时更新相关工作。

### 验收标准

**场景**：资产变更同步
- **假设**：某个资产被修改
- **当**：系统执行定时同步或收到事件触发
- **那么**：知识图谱更新该资产的状态和关联关系

**场景**：查看变更报告
- **假设**：系统完成一次同步
- **当**：查询方打开变更报告
- **那么**：系统列出新增、修改、删除的资产和关系

**场景**：确认系统发现的关系
- **假设**：变更报告中有系统自动发现的关系
- **当**：查询方审查并确认该关系
- **那么**：该关系标记为已确认，纳入正式图谱

### 业务规则

- V1.0 采用定时同步加事件触发方式
- 定时同步每天执行一次全量扫描
- 事件触发在文件变更时增量更新
- 同步结果生成变更报告供用户审查

### 依赖

- 依赖资产发现能力，需持续扫描资产变化
- 依赖工作流引擎，可配置同步触发策略
