# 数字资产定价

## 故事一：资产估值报告

### 用户故事
作为估值方，我想要生成数字资产的估值报告，以便在融资、商务、并购决策中有据可依。

### 验收标准

**场景**：生成资产估值报告
- **假设**：系统已收集足够的价格信号数据
- **当**：估值方选择资产或资产组合并请求生成报告
- **那么**：系统输出包含总资产估值、明细、方法说明、数据来源、置信度的报告

**场景**：导出报告
- **假设**：估值报告已生成
- **当**：估值方选择导出格式
- **那么**：系统生成 PDF 或 Markdown 格式的报告文件

**场景**：查看置信度分析
- **假设**：估值报告中包含置信度信息
- **当**：估值方查看某项资产的置信度
- **那么**：系统显示基于数据完整性和一致性的置信度计算说明

### 业务规则

- V1.0 采用成本法和市场法结合的估值方式
- 成本法基于薪酬绩效和采购数据提取的人力成本、外包成本、工具成本
- 市场法基于商务报价数据中同类方案的价格区间
- 每个资产估值包含估值金额、方法、置信度、估值日期
- 报告支持趋势分析和对比分析

### 依赖

- 依赖价格数据采集，需有足够的价格信号
- 依赖定价模型，需完成估值计算

## 故事二：定价数据整合

### 用户故事
作为数据使用方，我想要系统自动整合多来源的定价数据，以便直接获取高质量数据用于建模分析。

### 验收标准

**场景**：导入定价数据
- **假设**：数据使用方有交易系统的导出数据
- **当**：数据使用方上传 CSV 文件
- **那么**：系统校验数据格式并导入合规数据

**场景**：处理不合规数据
- **假设**：导入的数据中有格式不合规的记录
- **当**：系统完成数据校验
- **那么**：不合规数据标记为待审查，合规数据正常导入

**场景**：审查修正数据
- **假设**：有待审查的数据记录
- **当**：数据使用方审查并修正后重新提交
- **那么**：系统更新该记录状态为已导入

### 业务规则

- V1.0 提供 CSV 导入适配器
- 导入时自动校验数据格式
- 价格信号随时采集，估值计算按月执行
- 用户可随时触发手动重算，系统提示计算成本
- 每月执行一次全量重算，过期报告标记为历史版本

### 依赖

- 依赖数据源适配器，支持不同格式的数据接入
- 依赖估值计算引擎，定期执行重算
