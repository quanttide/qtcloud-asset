# 数字资产定价

## 产品定位

从交易过程中提取价格信号，建立定价模型，为数字资产和知识产权提供有据可依的估值。

## 核心价值

资产估值从主观判断变为数据驱动。不再拍脑袋回答公司值多少钱，而是基于真实交易数据科学建模。融资、商务、并购决策都有据可依。

## 价格数据采集

系统从各类交易过程中自动提取价格信号。V1.0 支持三类数据源。

商务报价数据。记录给客户的方案报价，包含报价金额、方案内容、涉及的技术资产、报价结果。通过对接 CRM 系统或手动录入获取。

采购数据。记录采购供应商服务的合同和费用，包含采购金额、服务内容、对应的内部资产、采购时间。通过对接采购系统或手动录入获取。

薪酬绩效数据。记录员工的薪酬和绩效评估，包含薪酬水平、负责的数字资产、绩效评级。通过对接 HR 系统或手动录入获取。

每个价格信号记录来源系统、采集时间、原始数据。确保数据可追溯。

## 数据源适配

每个数据源通过适配器接入。适配器声明支持的资源类型、字段映射、采集频率。

V1.0 提供 CSV 导入适配器，用户可导出交易系统的数据后导入。后续版本提供 API 适配器对接主流 CRM、采购、HR 系统。

导入时自动校验数据格式，不合规的数据标记为待审查。用户可审查并修正后重新导入。

## 定价模型

系统根据收集的价格信号建立资产定价模型。V1.0 采用成本法和市场法结合的方式。

成本法基于资产的历史投入估算价值。投入数据从薪酬绩效和采购数据中提取，包括人力成本、外包成本、工具成本。

市场法基于类似资产的市场交易价格估算价值。从商务报价数据中提取同类方案的价格区间。

每个资产的估值包含估值金额、估值方法、置信度、估值日期。置信度基于数据完整性和数据一致性计算。

## 报告与分析工具

提供定价报告功能。用户选择资产或资产组合，系统生成估值报告。

报告包含总资产估值、各资产估值明细、估值方法说明、数据来源、置信度分析。支持导出为 PDF 和 Markdown 格式。

分析工具支持趋势分析，查看资产估值随时间的变化。支持对比分析，比较不同资产或资产组合的估值差异。

## 数据更新策略

定价数据不是实时更新的。V1.0 采用月度更新策略，每月执行一次全量重算。

价格信号随时采集，但估值计算按月执行。月度报告记录计算日期，过期的报告标记为历史版本。

用户可随时触发手动重算，但系统会提示这会产生额外的计算成本。

## 使用场景

CEO 准备融资材料，打开系统选择所有知识产权相关的资产生成定价报告。报告给出总资产估值、各项资产估值明细、估值方法和数据来源。CEO 将报告附入融资材料，投资人看到有依据的估值更有信心。

商务团队给客户报价前，查看类似方案的历史报价和估值区间，参考这些数据制定更有竞争力的报价。
