# AI 交付约束

## 产品定位

定义数字资产验收规则，自动验证智能体交付物是否符合约束，让人机协作从检查细节升级为验收结果。

## 核心价值

验收标准显性化。不再依赖人工判断交付物是否合格，而是用规则引擎自动验证。管理者放心让 AI 承担更多工作，人工验收时间从小时级缩短到分钟级。

## 规则定义

用户通过 ISDL 统一数字资产描述语言定义验收规则。每个规则包含名称、描述、适用资产类型、验证条件、严重级别。

验证条件分为三类：格式检查验证文件结构和语法，内容检查验证必要章节和关键信息，关联检查验证与其他资产的一致性。

格式检查支持 Markdown 语法验证、必需头部字段检查、文件命名规范检查。内容检查支持必需章节检查、关键词检查、最小字数检查。关联检查支持引用链接有效性检查、与架构文档一致性检查。

严重级别分为错误和警告。错误级别不通过时资产标记为不合格，警告级别不通过时标记为建议修改。

## 规则模板

系统提供预定义的规则模板，覆盖主流资产类型。包括 PRD 模板、BRD 模板、架构文档模板、代码文档模板。

用户可基于模板创建自定义规则，修改检查项和阈值。模板本身可版本控制，支持 diff、回滚、fork。

## 验证引擎

智能体交付资产时系统自动触发验证引擎。验证引擎加载该资产类型的所有规则，逐条执行检查。

验证结果分为通过、不通过、跳过。通过表示满足所有规则，不通过表示有错误级别的失败，跳过表示规则不适用该资产。

每个检查结果包含规则名称、检查结果、失败原因、建议修复方式。不通过的项目高亮显示，方便快速定位问题。

## 报告与反馈

验证完成后生成验证报告。报告包含总分、通过规则数、不通过规则数、警告数、各项得分。

报告以结构化格式存储，支持 API 查询。智能体可通过 API 获取验证结果和自我修正。

用户可审查验证结果，对误报进行标记。系统记录误报信息，用于后续优化规则。

## 规则版本管理

规则支持版本控制，每个规则变更都有记录。版本包含版本号、变更内容、变更日期、变更人。

资产验证时记录使用的规则版本，确保验证结果可追溯。旧版本规则保留，不会影响已发布的规则。

规则可 fork 创建变体，适应不同团队或项目的差异化需求。

## 使用场景

产品经理让智能体生成用户管理模块的 PRD。智能体提交文档后，系统自动加载 PRD 规则模板进行验证。验证发现缺少安全性章节、性能指标未量化。验证报告指出这两个问题并给出建议。

智能体根据验证报告修改后重新提交，验证通过。产品经理只需审查验证通过的文档，不再需要逐字检查格式和结构。
