# 数字资产工作流

## 产品定位

工作流是数字资产契约的执行引擎，让用户定义的契约从静态文档变为可自动运行的跨平台流水线。

## 核心价值

跨平台自动化，一次定义自动在飞书、GitHub 等平台间流转数据。可视化编排，用户通过结构化配置定义工作流无需编写代码。可审计执行，每一步执行都有记录支持追溯和回放。

## 工作流定义

用户通过 ISDL 统一数字资产描述语言定义工作流，包含资产声明、转换规则、触发器、生命周期管理四个部分。资产声明让用户声明工作流涉及的平台资源，如飞书文档声明需要的字段和约束条件，GitHub Issue 声明需要的字段。转换规则让用户定义数据如何从一个平台转换到另一个平台，包括选择源资产和目标资产、配置转换动作、设置参数映射、定义执行条件。

触发器定义工作流何时启动。事件触发如飞书文档被打上特定标签时。定时触发如每天固定时间执行。手动触发用户手动启动。规则触发当资产成熟度评分达到阈值时自动触发，如成熟度超过 80 分时提示创始人或自动启动迁移。生命周期管理让用户管理工作流的运行状态，包括草稿、待审查、活跃、暂停、归档。

## 适配器生态

工作流通过适配器连接不同平台，每个适配器声明自己支持的能力包括支持的资源类型、支持的操作、支持的事件。用户在配置工作流时系统自动检查所选平台是否支持所需能力，避免配置无效的工作流。

## 执行监控

用户可查看每个工作流实例的当前状态，包括最近一次执行时间、执行结果、输入输出的摘要信息。工作流执行失败时自动暂停不会静默跳过，记录失败原因和上下文，通知相关负责人，提供恢复或降级选项。用户可查看工作流的完整执行历史，包括每一步的输入、输出、执行时间和操作人。

## 健康检查

系统定期检查工作流依赖的平台适配器是否可用，包括平台 API 是否可达、关键能力是否仍然存在。失败时自动暂停相关工作流并告警。

## 使用场景

飞书文档被打上同步标签时，自动创建 GitHub Issue 包含文档摘要和链接。每天定时从飞书多维表格拉取数据，生成报告并发布到指定频道。用户提交文档后自动触发审批流程，审批通过后同步到目标平台。

产品日志归档场景。创始人维护产品日志记录想法和决策，处理过的日志需要从工作区移动到归档站保持整洁。系统自动识别已提炼的日志，执行归档操作，清理空目录。

产品路线图生成场景。创始人积累的零散日志需要提炼为结构化的产品蓝图。系统收集日志，调用 AI 提炼核心想法、功能设计、关键决策，生成路线图文档，保持内容简洁结构化。
