# 数字资产契约

## 概念定义

数字资产契约是将隐性的工作流转化为显性的、可版本控制的、可迁移的数字资产契约。

## 核心价值

- 隐性工作流显性化：将隐式工作过程变成可视化的、可执行的数字资产契约流水线
- 可版本控制：契约本身可 diff、回滚、fork
- 可迁移：契约不依赖特定平台，可在飞书、GitHub 等平台间迁移

## 契约流水线

示例：飞书写文档 -> 提炼为 GitHub Issue

这个隐式过程被转化为一张可视化的、可执行的契约流水线，每一步都是显式定义的转换规则。

## 契约生命周期

- 草稿：契约定义阶段，可自由修改
- 待审查：契约分叉或修改后，需经审批
- 活跃：契约生效，开始执行
- 暂停：契约因错误或人工干预被阻断
- 归档：契约废弃或完成，保留历史记录

## 契约价值评分（CVS）

防止过度契约化，只对高分流程优先契约化：

- 高频（每天/每周多次）
- 跨人/跨团队协作
- 容易出错/需要精确控制
- 需要审计/合规追溯
- 数据资产价值高（影响收入/安全/合规）

低价值流程允许轻量级契约：只有触发 + 转换 + 输出结构，不做复杂治理。
