# 数字资产契约

## 概念定义

数字资产契约将隐性的工作流转化为显性的、可版本控制的、可迁移的数字资产契约。创始人用自然语言描述"把这个方案迁移给团队"，系统生成对应的契约配置，工作流引擎围绕契约自动执行治理操作。

## 核心价值

隐性工作流显性化，将隐式工作过程变成可视化的、可执行的数字资产契约流水线。契约本身可版本控制，支持 diff、回滚、fork。契约不依赖特定平台，可在飞书、GitHub 等平台间迁移。

## 契约流水线

飞书写文档提炼为 GitHub Issue，这个隐式过程被转化为一张可视化的、可执行的契约流水线，每一步都是显式定义的转换规则。

## 契约生命周期

契约从草稿开始，在定义阶段可自由修改。契约分叉或修改后进入待审查状态，需经审批。审批通过后契约生效变为活跃状态，开始执行。契约因错误或人工干预被阻断时进入暂停状态。契约废弃或完成后归档，保留历史记录。

## 成熟度评分

契约为资产附加成熟度评分，用于判断资产是否达到可共享状态。评分来源包括创始人为资料手动打分、系统自动统计引用次数和访问频率、团队成员的反馈和评分。成熟度评分与工作流触发器结合，当评分达到阈值时自动触发迁移流程。V1.0 先支持手动打分，V2.0 再支持自动统计。

## 治理语言记录

契约记录治理语言原文与 ISDL 翻译结果的对应关系。创始人用自然语言描述治理意图，系统记录原文并生成对应的 ISDL 工作流配置。V1.0 先支持简单模式匹配，人工确认后执行。

## 契约价值评分

契约价值评分防止过度契约化，只对高分流程优先契约化。高分流程的特征包括高频使用、跨人跨团队协作、容易出错需要精确控制、需要审计合规追溯、数据资产价值高影响收入安全合规。低价值流程允许轻量级契约，只有触发、转换、输出结构，不做复杂治理。
