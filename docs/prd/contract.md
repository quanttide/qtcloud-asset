# 数字资产契约

## 概念定义

数字资产契约是将隐性的工作流转化为显性的、可版本控制的、可迁移的数字资产契约。

## 核心价值

- 隐性工作流显性化：将隐式工作过程变成可视化的、可执行的数字资产契约流水线
- 可版本控制：契约本身可 diff、回滚、fork
- 可迁移：契约不依赖特定平台，可在飞书、GitHub、Jira 等平台间迁移

## 契约流水线

示例：飞书写文档 -> 提炼为 GitHub Issue -> 同步到 Jira

这个隐式过程被转化为一张可视化的、可执行的契约流水线，每一步都是显式定义的转换规则。

## 契约与治理范式

- 通用规则：制定通用规范和流程，使数字资产治理范式可复用
- 标准数字资产定义：定义和维护各类数字资产的标准和规范
- 契约即代码：契约用结构化格式（如 YAML/JSON）定义，可被机器解析和执行

## 契约的版本控制

- 内容寻址：通过哈希值标识契约版本
- 事件溯源：记录契约每次变更的事件，通过回放事件得到最新状态
- Schema 演进：契约遵循 ISDL（统一数字资产描述语言），版本控制的是 ISDL 而非底层存储

## 契约的生命周期

1. 草稿：契约定义阶段，可自由修改
2. 已发布：契约生效，开始执行
3. 已归档：契约废弃或完成，保留历史记录
