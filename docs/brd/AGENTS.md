# BRD 智能体指南

本文档指导 AI 智能体如何维护商业需求文档（BRD）。

## 文档阅读顺序

维护 BRD 时，智能体必须按以下顺序阅读文档。首先阅读 README.md 了解文档目录结构和定位，然后阅读 CONTRIBUTING.md 了解写作原则和审查清单，最后查看现有 BRD 文档了解当前业务需求。

## 必须遵守的规范

智能体在撰写或修改 BRD 时，必须严格遵守 CONTRIBUTING.md 中的所有原则。包括讲业务故事而非技术实现、使用业务语言避免技术术语、体现场景中的人物情感和动机、用具体例子说明需求、回答为什么这个需求重要。

## 文档格式

智能体撰写任何文档时都必须遵循[量潮科技写作格式标准](https://github.com/quanttide/quanttide-specification-of-business-entity/blob/v0.0.3/docs/format.md)。这个标准适用于所有文档写作场景，包括但不限于 BRD、PRD、技术文档、注释文档等。优先使用段落和标题组织内容，删除不必要的格式元素。能用列表就不用表格，能用文字就不用列表。同一概念全程使用相同名称。最多使用三级标题，分隔线全文最多 3 处。

## 质量检查流程

提交 BRD 前，智能体必须完成 CONTRIBUTING.md 中的审查清单自检。确认文档能让非技术人员看懂、包含真实的人物和场景、说明了业务动机和期待、具体到可以想象操作画面、清楚表达了为什么、避免了纯技术术语。

## 迭代更新原则

BRD 是活文档。智能体应主动与业务方沟通确认需求细节，根据反馈持续完善文档内容，记录需求演进过程，确保文档与实际业务保持一致。
