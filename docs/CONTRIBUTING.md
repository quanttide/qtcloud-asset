# 文档维护指南

本文档说明如何维护工作文档目录下的各类文档。

## 文档类型对比

商业需求文档（BRD）和产品需求文档（PRD）承担不同的职责。BRD 讲业务故事，关注为什么存在业务问题，用业务语言描述创始人的困扰、团队协作痛点、业务动机。BRD 面向业务方，文档有故事性、情感、场景画面。

PRD 讲产品方案，关注做什么功能，用产品语言描述产品定位、核心设计、功能清单。PRD 面向产品团队，文档功能导向、设计明确。关键区别在于 BRD 回答为什么存在业务问题，PRD 回答如何用产品解决问题。

BRD 从创始人的真实困扰出发，描述他希望摆脱重复劳动、希望知识资产能平滑转移。PRD 从产品角度出发，定义数字资产边界、核心功能设计、技术方案。先有 BRD 讲清楚业务痛点，才有 PRD 定义产品方案。

## 文档格式

所有文档必须遵循[量潮科技写作格式标准](https://github.com/quanttide/quanttide-specification-of-business-entity/blob/v0.0.3/docs/format.md)。优先使用段落和标题组织内容，删除不必要的格式元素。能用列表就不用表格，能用文字就不用列表。同一概念全程使用相同名称。最多使用三级标题，分隔线全文最多 3 处。
