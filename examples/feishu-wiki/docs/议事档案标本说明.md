# 议事档案标本说明

> 这是从飞书知识库「议事档案」中提炼出来的标本描述，用于确认业务理解是否正确。

## 标本是什么？

标本是「从真实资产中发现结构，再固化下来」的产品。

我们的范式是「发现而非发明」：
- 不是先设计好模型再去套资产
- 而是先扫描真实资产，从中挖掘模式，再固化成标本

## 议事档案的结构

```
议事档案
├── 首页（议事活动和编号规则）
└── 议题
    ├── 议事规则    # 目标、任务、行动
    ├── 议程        # 会议安排
    ├── 决策        # 选项、准则、权衡、决定
    ├── 计划        # 执行计划
    ├── 复盘研讨    # 反思、总结
    ├── 提案审计    # 提案评审
    ├── 澄清评估    # 评估、澄清
    ├── 谈判报告    # 谈判记录
    └── 决议        # 最终决定
```

## Workflow（能力编排流程）

> workflow 描述能力在什么状态下、按什么顺序被触发，是高内聚、低耦合的编排方式。

| 状态 | 说明 | 必选？ |
|------|------|--------|
| 议题创建 | 新议题按类型选择模板 | ✅ |
| 动议提出 | 在动议区填写核心内容（目标/选项/问题等） | ✅ |
| 附议扩散 | 相关人表态或补充 | 可选 |
| 决议形成 | 记录最终决定 | 可选，不是每议题都有 |
| 归档沉淀 | 存入知识库 | ✅ |

**状态流转：**
```
议题创建 → 动议提出 → 附议扩散 → 决议形成 → 归档沉淀
                      ↘ 无需决议 → 直接归档
```

**注意：** 辩论区在当前实践中不运转，已移除。

## 各议题类型的预期字段

### 议事规则
- 动议区（目标、@提及、行动项）
- 附议区（可选）

### 决策
- 元数据区（申请人、审议人）
- 动议区（选项列表）
- 准则（决策依据）
- 权衡（选项分析）
- 决定（最终选择）
- 附议区（各方意见）
- 决议区（最终决定，可选）

### 复盘研讨
- 问题清单
- 解决方案
- 个人心得
- 附议区（可选）
- 决议区（可选）

### 其他类型
议程、计划、提案审计、澄清评估、谈判报告、决议均有各自的标准字段。

## 标本验证的问题

验证标本时，发现以下问题：

| 问题 | 说明 |
|------|------|
| 元数据区 | 决策文档有「申请人、审议人」，已补充到标本 |
| 个人心得 | 复盘文档有「个人心得」，已补充到标本 |
| 决议区经常缺失 | 不是每议题都有决议，改为可选 |
| 辩论区全部为空 | 实践中不运转，从标本中移除 |

## 请确认

请确认以下几点：

1. **结构是否正确？** 议事档案下应该有这9种议题类型吗？有遗漏的吗？
2. **生命周期是否正确？** 实际工作中，走的是这5步吗？有补充的吗？
3. **各类型的预期字段是否准确？** 有没有缺或者多余的？
4. **「辩论区不运转」这个判断对吗？** 还是说只是暂时没用到？
